A · Full Canvas Replace
点击侧边栏 Marketplace 图标，主内容区完全替换为 marketplace 视图（同 Settings 的切换方式）。"← Projects" 按钮返回 repo 列表。最大化可用空间，4 列卡片网格。
Pros
  • 空间最大，显示最多卡片
  • 实现最简单，与现有设置切换模式一致
  • 最干净，无分心
Cons
  • 浏览 marketplace 时看不到 repos
  • 需要点击返回才能回到项目列表
Window size
900×600 (unchanged)
Nav pattern
Back button + sidebar icons
Implementation
Low complexity

B · Compact List Rows
与 A 相同的全替换模式，但技能以紧凑的列表行呈现（而非网格卡片）。每行显示图标、名称、作者、标签和安装按钮。单屏可显示更多技能，密度更高，适合快速浏览。
Pros
  • 密度最高，每屏显示 8-10 个技能
  • 快速扫描，信息更清晰
  • 更接近 Linear/Raycast 风格
Cons
  • 无法展示技能预览图或详细描述
  • 视觉吸引力不如卡片
Window size
900×600 (unchanged)
Layout
List rows

C · Horizontal Split
左侧 160px 保留 repo 列表（与当前 Skills Panel 打开时的列表栏类似），右侧显示 marketplace。可以选择 repo 后直接在右侧安装，无需切换视图。Repo selector 下拉框变为左侧列表点击选择。
Pros
  • 随时可见 repos，切换 repo 直接点击
  • 与现有 SkillsPanel 分栏风格一致
  • 不需要"返回"导航
Cons
  • marketplace 宽度压缩（只有 688px）
  • 3 列卡片略小
Window size
900×600 (unchanged)
Layout
160px + 688px split
Implementation
Low-Medium

D · Slide Drawer
Marketplace 从右侧滑入，覆盖约 70% 的窗口宽度。左侧 repo 列表依然可见但被半透明遮罩变暗。带有动画过渡效果。点击 × 或遮罩区域关闭。
Pros
  • 视觉上明确区分 "临时" 状态
  • 关闭后自动回到 repos
  • 动画过渡流畅感强
Cons
  • Drawer 宽度受限（约 620px）
  • 实现相对复杂（动画 + z-index）
Window size
900×600 (unchanged)
Pattern
Slide-over drawer
Implementation
Medium complexity

E · Tab Bar Navigation
将顶部 topbar 改为水平 Tab 导航："Projects | ★ Marketplace | Presets"。点击 tab 切换视图，侧边栏图标和 tab 双重导航并存。Presets 也作为 tab 出现，与 sidebar 形成层次化导航。
Pros
  • 所有主要视图一目了然
  • Presets 单独成 tab 比埋在 marketplace 里更突出
  • 直觉性强，与 web app 习惯一致
Cons
  • tab + sidebar 双重导航可能造成冗余
  • topbar 高度被 tab 占用，内容区略小
Window size
900×600 (unchanged)
Pattern
Horizontal tab bar
Implementation
Low-Medium

F · Command Palette Style
以搜索为核心。顶部是大号搜索框（带 ⌘K 提示），技能以结构化列表呈现，分 "已安装 / 全部" 两组。更偏 Raycast / Alfred 风格，键盘操作体验最佳。
Pros
  • 键盘操作体验最佳
  • 密度高，分组清晰
  • 极简风格，无视觉干扰
Cons
  • 无卡片封面图，视觉吸引力弱
  • 与当前 marketplace 卡片风格差异最大，改动最多
Window size
900×600 (unchanged)
Pattern
Search-first list
Implementation
Medium (UI redesign)

G · Sidebar Expansion Panel
点击 Marketplace 图标，侧边栏从 52px 展开为 280px 的面板，显示紧凑列表式 marketplace。主内容区（repo 列表）依然可见但被 overlay 遮暗。关闭面板回到 52px 侧边栏。
Pros
  • Repos 依然可见（可选择 repo 作为安装目标）
  • 与 VS Code 侧边栏展开模式相似
  • 不需要切换视图，状态连续
Cons
  • 展开后主内容区只有 620px
  • 卡片视图无法展示（空间不够），只能显示列表
Window size
900×600 (unchanged)
Pattern
Expanding sidebar panel
Implementation
Medium complexity

H · Window Expands (1100×660)
点击 Marketplace 图标时，窗口从 900×600 动态扩大至 1100×660，提供更宽敞的浏览空间。5 列卡片网格，每页显示更多技能。关闭 marketplace 时窗口缩回 900×600。
Pros
  • 空间最大，5 列展示最多内容
  • 接近原来独立窗口的使用体验
  • 不牺牲卡片尺寸
Cons
  • 窗口大小变化会破坏"紧凑 launcher"的感觉
  • 窗口动态 resize 实现稍复杂
  • 1100px 可能被屏幕边界截断
Window size
900→1100×660 (dynamic)
Pattern
Dynamic resize
Implementation
Medium (window resize)
